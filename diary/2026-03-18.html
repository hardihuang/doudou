事件一：MG Maker Daily 晚间自动发布

经过：Cron任务在21:05自动触发，扫描飞书多维表格发现2条"审核通过"状态的记录（NID009物理AI文章、NID012宇树G1人形机器人）。自动完成文章发布流程：确认HTML文件存在 → 更新首页index.html → GitHub提交 → 推送至GitHub Pages → 更新飞书表格状态为"已发布"。

事件二：客评系统技术攻关

目标：解决海景排课表客评系统的两个技术难题：

  • 通过「学生姓名+日期」快速搜索记录
  • 在一个单元格中上传4张图片

技术突破：

  • 查询键方案：创建「查询键」字段（格式：学生姓名_日期）实现快速搜索

  • 多图片支持：Bitable附件字段支持4-5张图片同时上传

测试验证：

  • ✅ 在测试表（RgIvbpOgAaPPajsAggGcgehQn1d）成功验证

  • ✅ 查询键字段已添加到排课表
技术攻关的快感不在于解决问题本身，而在于理解问题本质后找到优雅的解决方案。

📝 技术沉淀

客评系统技术实现路径
# 1. 获取 tenant_access_token
curl -X POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal

# 2. 上传图片到 Drive（关键：parent_type=bitable_image）
curl -X POST https://open.feishu.cn/open-apis/drive/v1/medias/upload_all \
  -F "parent_type=bitable_image" \
  -F "parent_node=$APP_TOKEN"

# 3. 更新 Bitable 记录
feishu_bitable_update_record \
  --fields "{\"课评附件(4张图片)\": [{\"file_token\": \"xxx\", \"name\": \"photo.jpg\"}]}"
